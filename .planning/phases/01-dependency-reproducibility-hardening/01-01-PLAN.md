<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream git host → build | A git ref the build resolves at install time; an upstream force-push/delete or branch-head move can change or remove the code the build pulls |
| PyPI registry → build | A version range the resolver picks from at install time; a new breaking major can be selected silently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering | `openhomedevice` git source (was moving `branch = remove-lxml-dependency`) | mitigate | Pin to immutable commit SHA `6e862a1022f59a21c57c501dcf040f81d12ebfaf` (DEP-01). A branch head can be force-moved; a SHA cannot. uv.lock records `?rev=SHA#SHA`. |
| T-1-02 | Tampering / DoS | All unbounded direct PyPI deps (`aiomqtt`, `paho-mqtt`, `websockets`, `httpx`, `requests`, etc.) | mitigate | Add next-major upper bounds (DEP-03) so a breaking major (e.g. paho-mqtt 2.x callback break, aiomqtt 3.x) cannot be resolved silently. |
| T-1-03 | Elevation / Tampering | `pyatv` git source (bare upstream commit on a repo the user does not own) | mitigate | Migrate to PyPI `pyatv==0.17.0` (immutable wheel). Removes the only residual non-user-owned git source; eliminates the dependency-confusion/force-push surface. |
| T-1-SC | Tampering | `pyatv==0.17.0` install (new PyPI resolution) | accept | RESEARCH.md Package Legitimacy Audit marks pyatv 0.17.0 APPROVED — established 8+ year project, active maintainer (postlund), released Jan 2026, migrating from an already-trusted pinned commit. No `[ASSUMED]`/`[SUS]`/`[SLOP]` packages → no blocking legitimacy checkpoint required. |
</threat_model>
